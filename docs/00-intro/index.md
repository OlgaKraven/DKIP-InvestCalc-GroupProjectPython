# InvestCalc — Инвестиционный аналитик ИС

Учебная мини-информационная система для оценки экономической эффективности внедрения информационных систем.

---

## 1. Назначение проекта

**InvestCalc** — это облегчённый аналитический инструмент, предназначенный для расчёта экономической эффективности проектов внедрения ИС.  
Он используется в учебных целях в рамках дисциплин ПИДИС и групповых проектов, а также может применяться как прототип в реальной аналитической деятельности.

Система позволяет рассчитывать:
- **TCO** — полную стоимость владения;
- **ROI** — рентабельность инвестиций;
- **Payback Period** — срок окупаемости;
- **анализ чувствительности (±20%)** — оценка устойчивости модели.

## 4. Ключевые функции InvestCalc

### 4.1. Расчёт экономических показателей
Ввод данных:
- CAPEX,
- OPEX,
- эффекты,
- период анализа (лет).

Система выполняет:
- TCO = CAPEX + OPEX × период,
- ROI = (суммарный эффект – TCO) / TCO,
- Payback Period = CAPEX / (эффект – OPEX).

---

### 4.2. Анализ чувствительности (±20%)
Автоматическое генерирование вариаций параметров:
- CAPEX × {0.8, 1.0, 1.2},
- OPEX × {0.8, 1.0, 1.2},
- (опционально) эффектов.

Выдача результатов в:
- таблице,
- JSON,
- графическом виде (в отчётах).

---

### 4.3. Поддержка сценариев
Сравнение вариантов:
- локальное внедрение (on-premises),
- облачное (SaaS / подписка),
- смешанная модель.

---

### 4.4. Генерация отчётов
Система формирует:
- HTML-отчёт (обязательно),
- PDF-отчёт (опционально).

Отчёты содержат:
- входные данные,
- результаты расчётов,
- чувствительность,
- рекомендации.

---

### 4.5. REST API (FastAPI)
Эндпоинты:
- `POST /api/v1/calculate`,
- `POST /api/v1/sensitivity`.

Поддерживаются:
- валидация входных данных,
- Pydantic-модели,
- Swagger UI,
- Redoc.

---

### 5. Архитектурная концепция

InvestCalc разработан по принципу разделения ответственности:

API (FastAPI)
↓
Service Layer (логика расчётов)
↓
Models (Pydantic)

Система stateless, легко контейнеризуется и поддерживает автоматические тесты.

Полное архитектурное описание:
- `docs/architecture.md`
- папка `docs/03-architecture`.
