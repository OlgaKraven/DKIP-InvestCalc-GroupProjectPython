# Предметная область проекта InvestCalc

## 1. Общая характеристика предметной области

Современные организации регулярно внедряют информационные системы (ИС), преследуя цели оптимизации процессов, снижения издержек, повышения производительности труда и улучшения качества обслуживания клиентов. Однако любое внедрение ИС связано с инвестициями, которые должны быть экономически обоснованы.

Расчёт экономической эффективности ИС является ключевой частью процесса принятия решений руководством. Стандартные подходы включают анализ капитальных затрат (CAPEX), операционных расходов (OPEX), совокупной стоимости владения (TCO), возврата инвестиций (ROI) и срока окупаемости (Payback Period). Для получения объективных результатов требуется унифицированная аналитическая модель и удобный инструмент автоматизации расчётов.

**InvestCalc** — это учебная мини-информационная система, предназначенная для оценки экономической эффективности проектов внедрения ИС на основе стандартных методик расчёта.

---

## 2. Объект автоматизации

Объектом автоматизации является **процесс экономической оценки ИС**, включающий:

- сбор исходных данных о затратах и эффектах;
- формирование аналитической модели затрат;
- вычисления ключевых финансовых показателей:
  - TCO (Total Cost of Ownership);
  - ROI (Return on Investment);
  - срок окупаемости (Payback Period);
- проведение анализа чувствительности параметров;
- формирование отчётности.

Система предназначена для использования студентами, аналитиками и менеджерами проектов, принимающими решения о внедрении ИС.

---

## 3. Основные понятия предметной области

### 3.1. CAPEX (Capital Expenditures)
Капитальные затраты на проект внедрения ИС:
- лицензии;
- оборудование;
- разработка и внедрение;
- обучение сотрудников при старте.

### 3.2. OPEX (Operational Expenditures)
Годовые операционные расходы на поддержку ИС:
- техподдержка;
- сопровождение;
- обновления;
- серверные мощности;
- SaaS-подписки (если применимо).

### 3.3. Эффекты от внедрения ИС
Экономические выгоды, выраженные в денежных единицах:
- сокращение трудозатрат;
- уменьшение количества ошибок;
- повышение скорости выполнения операций;
- снижение издержек.

### 3.4. Период анализа
Количество лет, на которое рассчитывается экономический эффект.

### 3.5. TCO (Total Cost of Ownership)
Полная стоимость владения ИС за период анализа:  
**TCO = CAPEX + Σ(OPEX по годам)**

### 3.6. ROI (Return on Investment)
Показатель рентабельности инвестиций:  
**ROI = (Эффекты * Период – TCO) / TCO**

### 3.7. Payback Period
Период окупаемости:  
**Сколько лет требуется, чтобы эффекты превысили стоимость внедрения.**

### 3.8. Анализ чувствительности
Метод, позволяющий понять, как изменятся показатели эффективности при изменении входных данных (±20% для CAPEX, OPEX, эффектов).

## 4. Границы предметной области

Границы системы включают:

### Внутренние процессы:
- ввод исходных данных;
- расчёт финансовых показателей;
- генерация таблиц и отчётов;
- анализ чувствительности;
- предоставление API-методов (REST).

### Внешние процессы:
- принятие решений менеджментом;
- обучение студентов методам оценки ИС;
- интеграция в учебные проекты.

Система **не** включает:
- ведение проектов внедрения;
- календарное планирование;
- оценку рисков;
- анализ архитектуры ИС как объекта внедрения.

---

## 5. Пользователи и заинтересованные стороны

### 5.1. Основные пользователи
- студенты и команды, выполняющие групповые проекты;
- преподаватели-проверяющие;
- младшие системные аналитики;
- проектные менеджеры.

### 5.2. Заинтересованные стороны
| Сторона | Интерес |
|---------|---------|
| Преподаватель | Проверка правильности расчётов и полноты документации |
| Студенты | Получить оценку инвестиций ИС и освоить методы анализа |
| Аналитики | Использовать как основу для прототипов в реальных проектах |
| Руководство организаций | Понимание окупаемости ИС |

## 6. Цели автоматизации

1. Упростить расчёт показателей эффективности внедрения ИС.
2. Предоставить универсальную модель расчёта.
3. Автоматизировать анализ чувствительности.
4. Создать учебный инструмент с API для командных проектов.
5. Обеспечить повторяемость и корректность результатов.
