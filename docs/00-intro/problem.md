# Бизнес-проблема, решаемая проектом InvestCalc

## 1. Введение в проблему

Организации всё чаще внедряют информационные системы (ИС) для повышения эффективности своей деятельности. Однако любое внедрение сопровождается значительными затратами: покупкой лицензий, оплатой разработки, инфраструктуры, сопровождения и обучения сотрудников.

Перед руководством встаёт ключевой вопрос:

> **Окупится ли внедрение информационной системы и является ли проект экономически целесообразным?**

На практике анализ окупаемости проводится нерегулярно, неполно или формально. Особенно это касается учебных проектов, малых компаний и внедрения простых ИС, где нет выделенных аналитиков.

Это приводит к ошибочным решениям, перерасходу бюджета и отсутствию прозрачности по экономическим эффектам.

---

## 2. Корневые причины проблемы

Основные причины, почему компании и учебные команды сталкиваются с трудностями при оценке экономической эффективности ИС:

### 2.1. Отсутствие стандартизированного инструментария
Каждый аналитик или студент использует собственные Excel-таблицы, что приводит к:
- несопоставимости результатов;
- разночтениям в расчётах;
- ошибкам в формулах.

### 2.2. Сложность расчётов для начинающих
Целый ряд базовых показателей (TCO, ROI, Payback) требует понимания:
- структуры затрат;
- финансовых методик;
- инвестиционного анализа.

Студентам и младшим аналитикам это даётся сложно без инструментальной поддержки.

### 2.3. Невозможность быстро проверить альтернативы
Чтобы сравнить несколько вариантов внедрения (например, On-Premises vs Cloud), нужно выполнить многократные расчёты вручную.

Это трудоёмко и приводит к ошибкам.

### 2.4. Отсутствие анализа чувствительности
Изменение ключевых параметров (±20% CAPEX, OPEX и эффектов) сильно влияет на результаты.

Без инструмента анализа чувствительности решения принимаются без учёта рисков.

### 2.5. Недостаточная прозрачность для руководства
Руководству сложно интерпретировать:
- в каких условиях проект окупается;
- насколько надёжны исходные данные;
- какой вклад в расчёты вносит каждый параметр.

---

## 3. Как проблема проявляется сейчас

В реальной практике мы наблюдаем следующие симптомы:

| Симптом | Последствия |
|---------|-------------|
| Расчёты ведутся вручную в Excel | Ошибки, несогласованность версий |
| Нет единой модели расчётов | Каждый студент/аналитик создаёт свою модель |
| Нет анализа чувствительности | Не оцениваются риски и устойчивость модели |
| Нет API и автоматизации | Невозможно интегрировать в приложения/сервисы |
| Нет отчётности | Руководитель не может быстро увидеть выводы |

---

## 4. Почему проблема значима

### Для образовательных целей:
- студенты не понимают взаимосвязь CAPEX, OPEX, TCO, ROI и Payback;
- командные проекты страдают от отсутствия унифицированных расчётов;
- преподавателю сложно проверять большое количество Excel-моделей.

### Для бизнеса:
- решения принимаются «на ощущениях», а не на основе данных;
- можно ошибочно инвестировать в нерентабельную ИС;
- сложно прогнозировать экономические риски внедрения.

---

## 5. Требуемое решение проблемы

Для решения обозначенных проблем необходим инструмент, который:

### 5.1. Стандартизирует расчёты
Единая математическая модель:
- TCO,
- ROI,
- Payback Period,
- Анализ чувствительности.

### 5.2. Исключает ошибки
Система должна автоматически выполнять расчёты и проверять входные данные.

### 5.3. Упрощает многовариантный анализ
Можно быстро создать:
- локальный сценарий,
- облачный сценарий,
- альтернативные модели.

### 5.4. Автоматизирует анализ чувствительности
Вариации параметров должны считаться автоматически (по умолчанию ±20%).

### 5.5. Формирует отчёты
Для преподавателя, аналитика и руководителя:
- в текстовом виде (MD, HTML, PDF),
- с графиками,
- с таблицами.

### 5.6. Предоставляет API
Чтобы:
- выполнять расчёты через интеграции,
- использовать в фронтенд-приложениях,
- подключать в учебные проекты.

---

## 6. Ценность решения

Реализация InvestCalc позволит:

### Для студентов:
- глубоко понимать финансовые аспекты ИС,
- использовать готовую модель расчётов в проектах,
- быстрее выполнять задания ПИДИС.

### Для преподавателей:
- проверять работы быстрее,
- получать стандартизированные расчёты,
- снизить количество ошибок у студентов.

### Для бизнеса:
- принимать более обоснованные решения,
- оценивать риски внедрения,
- анализировать альтернативные варианты внедрения ИС.
