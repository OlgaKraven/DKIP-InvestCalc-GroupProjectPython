# Примеры фрагментов отчёта по проекту InvestCalc

Этот документ содержит примеры текстовых фрагментов, которые студенты могут использовать при подготовке итогового отчёта по учебному проекту InvestCalc.
Фрагменты можно адаптировать под конкретную предметную область и сценарий.

---

## 1. Введение (пример)

> В рамках учебной проектной работы была разработана информационная система InvestCalc, предназначенная для оценки экономической эффективности внедрения информационных систем.
> Система реализует расчёты показателей TCO (Total Cost of Ownership), ROI (Return on Investment) и срока окупаемости проекта (Payback Period), а также предоставляет инструменты анализа чувствительности ключевых параметров.
> Целью разработки является формирование у студентов компетенций в области аналитики, архитектурного проектирования, разработки REST API и применения DevOps-практик на примере учебного прототипа.

---

## 2. Описание предметной области (пример)

> Информационная система InvestCalc оперирует экономическими параметрами, которые применяются при оценке эффективности внедрения ИС: капитальные затраты (CAPEX), операционные расходы (OPEX), годовые эффекты и стоимость владения.
> Пользователь формирует один или несколько сценариев развития и сравнивает показатели эффективности.
> Данный подход позволяет быстро оценить, насколько выгодно внедрение ИС, а также определить риски и чувствительность решений к изменению параметров.

---

## 3. Постановка проблемы (пример)

> В организациях нередко возникает необходимость обосновать затраты на внедрение ИТ-решения. При отсутствии удобных инструментов сравнения сценариев сложно принимать обоснованные управленческие решения.
> InvestCalc решает эту проблему, предоставляя простой, наглядный и воспроизводимый инструмент расчётов, который помогает оценить экономическую эффективность проекта даже на ранних этапах.

---

## 4. Требования и сценарии использования (пример)

> В ходе проектирования были определены ключевые функциональные требования, включающие расчёт TCO, ROI, Payback Period и анализ чувствительности ±20%.
> Построенные сценарии использования рассматривают действия пользователя при вводе данных, расчёте показателей, сравнении сценариев и получении результатов.
> Эти сценарии легли в основу проектирования API и тестовых примеров.

---

## 5. Архитектура системы (пример)

> Архитектура InvestCalc построена по принципу разделения ответственности:
>
> * FastAPI обрабатывает HTTP-запросы и обеспечивает REST API,
> * доменная логика реализована в отдельном модуле,
> * модели данных описаны через Pydantic,
> * контейнеризация реализована с помощью Docker.
>   Архитектурные решения зафиксированы в ADR-документах, а структура проекта документирована в виде C4-диаграмм (Context, Container, Component).

---

## 6. Модель данных (пример)

> Модель данных включает сущности, описывающие параметры расчётов: CAPEX, OPEX, период анализа, денежные потоки и экономические эффекты.
> ER-диаграмма и предметная модель показывают структуру данных и связи между ними.
> Несмотря на то что в учебной версии данные не хранятся в БД, модель данных используется как основа для валидации входных значений и расчётов.

---

## 7. API (пример)

> API реализовано на базе FastAPI и предоставляет REST-эндпоинты для расчётов TCO, ROI, Payback Period и анализа чувствительности.
> API документировано автоматически через OpenAPI, Swagger UI (`/docs`) и ReDoc (`/redoc`).
> Реализована валидация входных данных, обработка ошибок и возврат корректных HTTP-кодов.

---

## 8. Тестирование (пример)

> В проекте применяется комбинация модульных и интеграционных тестов.
> Модульные тесты проверяют корректность математических расчётов, а интеграционные — взаимодействие API и доменной логики.
> API-тесты выполняются с использованием `httpx.AsyncClient`, что позволяет имитировать реальные запросы.
> Все тесты автоматически запускаются в CI-пайплайне при каждом коммите в ветку `main` и при создании Pull Request.

---

## 9. CI/CD (пример)

> В проекте настроен конвейер CI/CD на GitHub Actions, который включает:
>
> * матричное тестирование на Python 3.10–3.12,
> * линтинг кода (ruff, black),
> * анализ безопасности (bandit, safety),
> * сборку документации MkDocs,
> * создание и публикацию Docker-образа в GitHub Container Registry.
>   Такой подход обеспечивает стабильность, повторяемость и прозрачность всех процессов разработки.

---

## 10. Docker и развёртывание (пример)

> Развёртывание InvestCalc осуществляется с помощью Docker.
> Docker-образ автоматически публикуется в GHCR и может быть запущен локально или на сервере с минимальной конфигурацией.
> Поддерживается также развёртывание через docker-compose.
> Это позволяет студентам получить практический опыт работы с контейнерами и современными DevOps-подходами.

---

## 11. Результаты и выводы (пример)

> В результате работы был создан учебный прототип InvestCalc, позволяющий проводить расчёты ключевых показателей эффективности ИС.
> Система реализована в виде REST API с чёткой архитектурой, документирована и покрыта тестами.
> Настроенный CI/CD и Docker-окружение обеспечивают воспроизводимость и высокое качество разработки.
> Проект может быть использован как пример комплексной учебной информационной системы, демонстрирующей современные практики разработки и сопровождения.

---

## 12. Краткая итоговая формулировка (готовая для отчёта)

> Проект InvestCalc успешно реализован и включает полный цикл:
> анализ требований → архитектурное проектирование → реализация → тестирование → CI/CD → развёртывание → отчётность.
> Полученный прототип позволяет проводить экономический анализ ИТ-проектов, демонстрируя современные подходы к разработке информационных систем.
