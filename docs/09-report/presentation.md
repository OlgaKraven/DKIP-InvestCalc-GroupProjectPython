# Presentation — Презентация проекта InvestCalc

Данный документ описывает структуру и требования к презентации при защите проекта.

---

## 1. Общие рекомендации

- 7–12 слайдов;
- минимальный текст — максимум визуализации;
- крупные заголовки, читабельность с экрана;
- единый стиль оформления.

---

## 2. Структура презентации

### Слайд 1. Титульный
- название проекта;
- логотип команды (опционально);
- участники;
- преподаватель;
- год.

### Слайд 2. Проблема
- почему компаниям нужно рассчитывать эффективность ИС;
- примеры бизнес-ситуаций.

### Слайд 3. Цели проекта
- что решает InvestCalc;
- ключевые задачи.

### Слайд 4. Предметная область
- диаграмма предметной модели (краткая);
- основные сущности.

### Слайд 5. Функционал
- расчёт TCO/ROI/Payback;
- анализ чувствительности;
- сравнение сценариев;
- создание отчётов.

### Слайд 6. Архитектура
- C4 Container Diagram (мершайд);
- структура сервисов.

### Слайд 7. Реализация
- FastAPI;
- Pydantic;
- структура src/.

### Слайд 8. Результаты
- примеры расчётов;
- графики чувствительности.

### Слайд 9. Docker и DevOps
- docker-compose up;
- CI/CD;
- healthchecks.

### Слайд 10. Выводы
- окупаемость;
- ограничения;
- направления развития.

---

## 3. Требования к качеству

- одинаковые шрифты;
- диаграммы читаемые;
- минимум текста → максимум смысла;
- не перегружать расчётными формулами.
