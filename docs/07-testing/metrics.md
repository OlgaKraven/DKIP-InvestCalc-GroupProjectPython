# Metrics — Метрики качества тестирования InvestCalc

Документ описывает, какие метрики используются для оценки качества тестирования и состояния проекта.

---

## 1. Цели метрик

- Оценить полноту тестирования.
- Отслеживать прогресс исправления дефектов.
- Обосновать готовность проекта к сдаче/презентации.

---

## 2. Базовые метрики

### 2.1. Количество тестов

- Общее количество автотестов (pytest).
- Разбивка по уровням:
  - Unit-тесты (service)
  - API-тесты
  - Тесты чувствительности

## 2.2. Процент пройденных тестов

- `passed_tests / total_tests * 100%`

Цель — 100% на момент сдачи проекта.

---

## 3. Дефекты

### 3.1. Количество найденных дефектов

- Всего заведено багов.
- По Severity:
  - Blocker
  - Critical
  - Major
  - Minor
  - Trivial

### 3.2. Количество открытых дефектов

- На момент защиты не должно быть непринятых/необработанных Critical/Blocker.

---

## 4. Покрытие кода (опционально)

Если использовать coverage:

- Покрытие модулей `services/invest_service.py` должно быть максимально (условно ≥ 80%).
- Покрытие API — ниже, но основные ветки должны быть проверены.

---

## 5. Метрики CI/CD

- Количество успешных/проваленных запусков pipeline.
- Среднее время выполнения тестов.
- Количество коммитов, нарушивших тесты.

---

## 6. Отчётность по метрикам

Рекомендуется:

- Вести табличку (Markdown/Excel) с динамикой:
  - Неделя 1, 2, 3, 4.
  - Кол-во тестов.
  - Кол-во багов.
  - Покрытие (если используется).
