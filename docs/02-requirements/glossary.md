## Глоссарий проекта InvestCalc

Документ содержит набор терминов, используемых в проектной документации, архитектуре, API и отчётной части системы «InvestCalc — Инвестиционный аналитик ИС».  
Глоссарий необходим для единообразного понимания всех ключевых понятий.

---

## 1. Финансовые термины

### CAPEX (Capital Expenditures)
Капитальные затраты на внедрение ИС: покупка лицензий, оборудования, обучение, разработка.

### OPEX (Operational Expenditures)
Годовые операционные расходы: поддержка, сопровождение, подписка, обновления, серверные мощности.

### TCO (Total Cost of Ownership)
Полная стоимость владения ИС за указанный период.  
Формула:  
**TCO = CAPEX + OPEX × период анализа**

### ROI (Return on Investment)
Рентабельность инвестиций — показатель эффективности вложений.  
Формула:  
**ROI = (Суммарный эффект − TCO) / TCO**

### Payback Period
Срок окупаемости проекта — количество лет, через которое эффекты от внедрения покроют инвестиции.

### Экономический эффект
Положительное денежное влияние внедрения ИС на бизнес: уменьшение затрат, рост производительности, снижение ошибок.

### Период анализа
Количество лет для расчёта TCO, ROI и Payback.

## 2. Термины предметной области

### Информационная система (ИС)
Совокупность программных и технических средств, обеспечивающих сбор, обработку и хранение данных.

### Эффективность ИС
Степень достижения системой цели (экономия, ускорение процессов, сокращение затрат).

### Сценарий локальной инфраструктуры
Вариант внедрения ИС на собственных серверах.

### Сценарий облачной инфраструктуры
Вариант внедрения ИС через SaaS / подписку.

### Анализ чувствительности
Метод анализа влияния изменения ключевых параметров (CAPEX, OPEX, эффектов) на итоговые показатели.

### Рекомендация
Вывод системы о выгодности или невыгодности проекта.

## 4. Термины разработки (Backend)

### FastAPI
Быстрый веб-фреймворк Python для построения REST API.

### REST API
Способ взаимодействия сервисов через HTTP и JSON.

### Эндпоинт
URL-адрес, к которому обращается клиент (например, `/api/v1/calculate`).

### Валидация данных
Проверка входных данных на корректность (тип, диапазон, ограничения).

### Pydantic
Библиотека Python для работы с типизированными моделями.

### Модель данных (Schema)
Структура входного и выходного JSON.

### Сервисный слой (Service Layer)
Модуль, содержащий бизнес-логику расчётов.

### Бизнес-логика
Правила, формулы и методы, определяющие работу аналитических функций.

### Uvicorn
ASGI-сервер для запуска FastAPI.
