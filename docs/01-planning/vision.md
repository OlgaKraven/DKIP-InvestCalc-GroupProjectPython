# Project Plan — План проекта InvestCalc

## 1. Этапы проекта

| Этап | Содержание | Артефакты |
|------|------------|-----------|
| 1. Постановка задачи | Анализ предметной области, проблемы, данных | domain.md, problem.md |
| 2. Моделирование | Требования, сценарии, архитектура | use-cases, C4, ADR |
| 3. Разработка | API, сервисы, тесты | src/, tests/ |
| 4. DevOps | Docker, CI/CD, инструкции | devops/ |
| 5. Документация | MkDocs сайт, отчёты | docs/, report |
| 6. Презентация | Визуализация результата | report/presentation.md |

---

## 2. Роли в проекте

- Тимлид
- Разработчик API
- Разработчик модели расчётов
- DevOps-инженер
- Технический писатель

---

## 3. Методология

- Канбан-доска (GitHub Projects)
- Issues + Pull Requests
- Code Review

---

## 4. Риски (топ-5)

| Риск | Вероятность | Влияние | План реакции |
|------|-------------|---------|--------------|
| Недостаток времени | Средняя | Высокое | Декомпозиция задач |
| Плохая коммуникация | Средняя | Среднее | Стендапы |
| Ошибки в расчётах | Низкая | Высокое | Тесты |
| Проблемы с Docker | Средняя | Среднее | Инструкции DevOps |
| Неполная документация | Средняя | Высокое | MkDocs + контроль |

---

## 5. Контроль качества

- тестирование (pytest),
- CI/CD проверки,
- проверка покрытий (опционально),
- валидация входных данных.
