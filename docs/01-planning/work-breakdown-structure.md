# Work Breakdown Structure (WBS)

Проект: **InvestCalc — Инвестиционный аналитик ИС**  
Тип проекта: учебный, командный (2–4 человека)  
Версия: 1.0

---

## 1. Подготовительный этап
### 1.1. Организация команды
- Анализ доступных ролей (Team Lead, Backend, QA/DevOps, UX/UI).
- Распределение ответственности.
- Организация канала коммуникаций (Discord/Telegram).

### 1.2. Подготовка репозитория
- Создание репозитория GitHub.
- Развёртывание стартовой структуры папок.
- Первичный README и лицензия.

### 1.3. Настройка среды
- Установка Python/C##/FastAPI/.NET (в зависимости от варианта).
- Установка poetry/pipenv/venv.
- Настройка IDE (VS Code / PyCharm / Rider).
- Базовые команды Git.

---

## 2. Аналитика и требования
### 2.1. Анализ предметной области
- Описание «клиента» и бизнес-проблемы.
- Мотивация внедрения InvestCalc.

### 2.2. Сбор требований
- User Stories (первичный набор).
- Use Cases (основные сценарии).
- Определение нефункциональных требований.

### 2.3. Формирование ТЗ
- Структура документа ТЗ.
- Разделы: назначение, состав функций, интерфейсы, ограничения, риски.

---

## 3. Архитектура
### 3.1. Техническая архитектура
- Создание C4-модели (Context, Container, Component).
- Модульная схема приложения.

### 3.2. Модель данных
- Логическая модель данных.
- Схема хранения JSON-файлов.

### 3.3. Проектирование API
- Структура `/api/v1`.
- Документирование эндпоинтов.

### 3.4. UI/UX-прототип
- Карта экранов (Screen map).
- Прототипы и пользовательские сценарии.

---

## 4. Реализация (Backend)
### 4.1. Модели данных
- Pydantic / DTO.
- Модели сценариев и расчётов.

### 4.2. Бизнес-логика
- Модуль расчётов (TCO, ROI, Payback).
- Анализ чувствительности ±N%.
- Работа с JSON-хранилищем.

### 4.3. Реализация API
- Эндпоинты:
  - `/calc`
  - `/sensitivity`
  - `/scenarios` (GET/POST/GET{id})
- Обработка ошибок.

### 4.4. Тестирование API вручную (черновое)
- Swagger.
- Тест кейсы в виде таблицы.

---

## 5. Тестирование и качество
### 5.1. Тестовая документация
- Test Plan.
- Test Cases.
- Bug Reports.
- Regression Checklist.

### 5.2. Автотесты
- Unit tests для расчётов.
- Sensitivity tests.
- API-интеграционные тесты.

### 5.3. Code Review
- Pull-Request шаблоны.
- Правила оформления коммитов.

---

## 6. DevOps и эксплуатация
### 6.1. Контейнеризация
- Dockerfile.
- docker-compose (если требуется).

### 6.2. Мониторинг
- Healthchecks.
- Логирование.

### 6.3. Документация эксплуатации
- Deployment manual.
- Backup & Restore.

### 6.4. CI/CD
- GitHub Actions workflow (минимальный).

---

## 7. Отчётность и защита
### 7.1. Отчёты
- Отчёт по проекту (group project).
- Отчёт по учебной практике.

### 7.2. Индивидуальные дневники
- Документирование личного вклада.

### 7.3. Подготовка презентации
- Слайды.
- Демонстрация API.
- Выводы и показатели эффективности.

---

## 8. Завершение проекта
### 8.1. Финальная проверка репозитория
- Полнота документации.
- Соответствие Roadmap.
- Работоспособность API через Docker.

### 8.2. Итоги и самооценка
- Анализ качества продукта.
- Обсуждение улучшений.
