## 2.2. Нефункциональные требования (NFR)

### 2.2.1. Требования к надежности
- устойчивость к ошибкам  
- обработка исключений  
- защита от некорректного ввода  

### 2.2.2. Требования к производительности
- время отклика API ≤ …  
- поддержка N одновременных пользователей  

### 2.2.3. Требования к безопасности
- JWT/OAuth2  
- хранение паролей только в хэшированном виде  
- ролевая модель доступа  

### 2.2.4. Требования к интерфейсу (UI/UX)
- наличие адаптивной вёрстки  
- доступность интерфейсов  
- единый стиль оформления  

### 2.2.5. Требования к данным
- структура БД;  
- ограничения, индексы;  
- форматы данных;  
- объём и частота изменений.  

### 2.2.6. Ограничения
- технологии;  
- языки программирования;  
- ограничение на браузеры;  
- хостинг;  
- совместимость.
