# Pull Request — Шаблон
*(Стандарт корпоративного оформления Pull Request для учебных и реальных проектов)*

---

## 1. Описание изменений
Кратко опишите, что было сделано в рамках этого PR.

**Пример:**
- Добавлен модуль расчёта ROI.
- Исправлена логика валидации CAPEX.
- Добавлен новый API endpoint `/api/projects/{id}/calculate`.
- Улучшен UI формы расчёта.

---

## 2. Тип изменений
Отметьте **один или несколько** вариантов:

- [ ] Bug fix (исправление ошибки)
- [ ] New feature (новый функционал)
- [ ] Refactoring (изменение внутренней логики без новых фич)
- [ ] Documentation update
- [ ] Tests
- [ ] DevOps / CI / Docker
- [ ] Architecture change / ADR update

---

## 3. Связанные задачи / Issue links
Укажите связанные задачи, чтобы GitHub автоматически закрыл issue при мерже:

```

Closes ##<номер>
Fixes ##<номер>
Relates to ##<номер>

```

---

## 4. Подробное описание
Опишите ключевые детали:

- что именно реализовано;
- какие файлы изменены;
- архитектурные решения;
- добавленные пакеты;
- изменения API / схемы БД;
- пересмотренные ADR.

---

## 5. Скриншоты, схемы, примеры (если UI или API)
Добавьте по необходимости:

- UI скриншоты,
- ответы API,
- диаграммы (Mermaid),
- фрагменты JSON.

**Пример:**
```json
{
  "projectId": 1,
  "roi": 27.5,
  "payback": 12
}
```

---

## 6. Чеклист автора (Self-check)

### Функциональность

* [ ] Код работает корректно.
* [ ] Все функции протестированы вручную.
* [ ] Нет временных заглушек или тестового кода.
* [ ] Проверены ошибки и крайние случаи.

### Тестирование

* [ ] Добавлены или обновлены unit tests / integration tests.
* [ ] Все тесты проходят (`dotnet test` / `pytest` / `npm test`).
* [ ] Проверено форматирование кода (`black`, `flake8`, `eslint`, `dotnet format`).

### Качество кода

* [ ] Код соответствует принятому стилю.
* [ ] Имена переменных, методов и классов понятны.
* [ ] Логика разделена по слоям (Controller / Service / Repository).
* [ ] Нет дублирования (DRY).
* [ ] Нет "магических чисел".

### Безопасность

* [ ] Нет утечек токенов / ключей / паролей.
* [ ] Добавлена или проверена валидация входных данных.
* [ ] Обработаны возможные исключения.

### Инфраструктура

* [ ] Обновлены Dockerfile / docker-compose (если требовалось).
* [ ] Обновлены GitHub Actions / CI workflows (если требовалось).
* [ ] Документация актуализирована.
